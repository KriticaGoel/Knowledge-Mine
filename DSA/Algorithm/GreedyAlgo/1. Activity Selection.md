## Pattern 2: Activity Selection

## Problem 1:

You are given two arrays:

start[] = {1, 3, 0, 5, 8, 5}
finish[] = {2, 4, 6, 7, 9, 9}

Each pair represents an activity.

Only one activity can be performed at a time.

Return the maximum number of activities that can be completed.

Expected Output : 4 
(1,2)  (3,4)  (5,7) (8,9)

### Solution:

This is an **activity problem** and **no overlapping allowed** need to find **max no of activity** so pattern 2 satisfied

Validate it's a Greedy problem or not

### Step 1: Is it asking for a minimum or maximum?

Yes we need to find maximum number of activities that can be completed.

### Step 2 : Can I sort the input?
Yes, we can short the input on bases of finish time in ascending order so that we can pick earlier finish

### Step 3 : Can I repeatedly pick the "best" option?
 once job finished we can take another job if starting time>= finish time else skip the job by that way we can cover can be run one after another

### Step 4 : Will that choice remain valid forever?
Choosing the activity that finishes earliest leaves the largest remaining time interval for future activities. 
Any activity that finishes later cannot increase the number of activities that fit afterward.

### Step 5 : Can I prove that taking the local optimum leads to the global optimum?
Suppose an optimal solution does not choose the activity with the earliest finish time.

Replace its first chosen activity with the earliest-finishing activity.

Since the replacement finishes no later than the original one, every remaining activity that was compatible before is still compatible afterward.

Therefore, the total number of activities does not decrease.

Hence, there always exists an optimal solution that starts with the earliest-finishing activity.

Therefore, repeatedly choosing the earliest-finishing compatible activity is optimal.

Greedy Algorithm is best fit.

### Coding
```java
public class Problem1 {

    static void main(String[] args) {


        int start[] = {1, 3, 0, 5, 8, 5};
        int finish[] = {2, 4, 6, 7, 9, 9};
        int maximumActivityCount=1;

        //Edge case if array is empty
        if (start.length == 0) {
            System.out.println("maximumActivity is 0");
            return;
        }

        //Pair each start and finish time into an Activity
        Activity[] activities = new Activity[start.length];

        for  (int i = 0; i < activities.length; i++) {
            activities[i] = new Activity(start[i], finish[i]);
        }

        // Sort the array as per finish time in asc order
        Arrays.sort(activities, Comparator.comparingInt(a -> a.finishTime));

        //Select the first activity.
        int lastFinish = activities[0].finishTime;

       //Iterate through the remaining activities.
        for(int i = 1; i < activities.length; i++) {
            //If an activity starts after or when the last selected one finishes, select it.
            if(lastFinish <= activities[i].startTime) {
                //Update last finish to compare with next start time
                lastFinish=activities[i].finishTime;
                //Count the selected activities.
                maximumActivityCount++;
            }
        }
        System.out.println(maximumActivityCount);

    }
}

class Activity{
    final int startTime;
    final int finishTime;

    Activity(int startTime, int finishTime) {
        this.startTime = startTime;
        this.finishTime = finishTime;
    }
}

```

### Time Complexity

Sorting : O(n log n)
Traversal : O(n)
Overall : O(n log n)

### Space Complexity
 O(n)
 
## Problem 2: Maximum Meetings in One Room

Problem

There is one meeting room.

Each meeting has:

* Start time
* End time

Find the maximum number of meetings that can be scheduled.

Start = [1,3,0,5,8,5]

End   = [2,4,6,7,9,9]

Expected output 4

Solution:

### Step 1 : Is it asking for a minimum or maximum?
Yes. The objective is to maximize the number of meetings that can be scheduled in a single room without overlaps. 
Since we are optimizing a quantity, this is an optimization problem.
### Step 2 : Can I sort the input?
Yes. I will sort meetings by finish time in ascending order because finishing earlier leaves the room available sooner, creating the maximum opportunity to schedule additional meetings.
### Step 3 : Can I repeatedly pick the "best" option?
Yes.

The greedy choice is:

Always select the meeting that finishes earliest among the remaining meetings.

After selecting it, ignore every meeting that overlaps and repeat the same decision.

### Step 4 : Will that choice remain valid forever?

Suppose we have:

Meeting A : 1 - 2

Meeting B : 1 - 5

Which one should we choose?

If we choose

1 - 5

the room is occupied until 5.

If we choose

1 - 2

the room becomes free at 2.

Now we can still schedule meetings from 2 onwards.

Therefore,

choosing the meeting that finishes earliest can never reduce our future choices.

That is why the greedy decision never needs to be changed.

### Step 5 : Can I prove that taking the local optimum leads to the global optimum?

Suppose we have two choices:

Meeting A

1 - 2
Meeting B

1 - 5

If I choose A,

the room becomes available at

2

If I choose B,

the room becomes available at

5

Since

2 < 5

every meeting that can be scheduled after B

can also be scheduled after A.

In fact,

A may allow additional meetings before 5.

Therefore,

choosing the earliest finishing meeting never makes the solution worse.

This is called the greedy-choice property.

## Code

```java
public class Problem2 {

    static void main(String[] args) {

        int[] start = {1,3,0,5,8,5};

        int[] end = {2,4,6,7,9,9};

        if(start.length ==0) {
            System.out.println("Maximum job is 0");
            return;
        }
        //sorted on the basis of finish time in asc order
        Meeting[] meetings = new Meeting[start.length];

        for (int i = 0; i < start.length; i++) {
            meetings[i] = new Meeting(start[i], end[i]);
        }

        Arrays.sort(meetings, Comparator.comparingInt(a->a.finishTime));

        int lastMeetingFinished = meetings[0].finishTime;
        System.out.println("start job "+meetings[0].startTime +" Finished job  "+meetings[0].finishTime);

        int maximumNumberOfMeeting =1;
         for(int i=1;i<meetings.length;i++){
             if(meetings[i].startTime >= lastMeetingFinished){
                 lastMeetingFinished = meetings[i].finishTime;
                 maximumNumberOfMeeting++;
                 System.out.println("start job "+meetings[i].startTime +" Finished job  "+meetings[i].finishTime);

             }
         }

        System.out.println("maximum number of meetings that can be scheduled "+maximumNumberOfMeeting);
    }
}

class Meeting {
    final int startTime;
    final int finishTime;
    public Meeting(int startTime, int finishTime) {
        this.startTime = startTime;
        this.finishTime = finishTime;
    }
}
```

Time complexity O(n log n)

Space Complexity O(n)

## Problem 3 : Non-overlapping Intervals

Problem

Given intervals

[[1,2],[2,3],[3,4],[1,3]]

Return the minimum number of intervals to remove so that the remaining intervals are non-overlapping.

Expected output 1

Solution:

### Step 1 : Is it asking for a minimum or maximum?
Yes, The objective is to minimize the no. of intervals to remove so that remaining intervals are non-overlapping.
Since we are optimizing the quantity that why it's an optimizing problem.

### Step 2 : Can I sort the input?
Sort intervals by end time so that when two intervals overlap, we keep the one that finishes first. Keeping the earlier finishing interval leaves the maximum room for future non-overlapping intervals.
### Step 3 : Can I repeatedly pick the "best" option?
Yes, greedy approach is if next start time is less than current finish time.
Keep the earliest finishing interval. 

### Step 4 : Will that choice remain valid forever?
Keeping the interval that finishes earliest leaves the largest remaining timeline for future intervals. Therefore, this decision never reduces the number of intervals that can still be kept.
### Step 5 : Can I prove that taking the local optimum leads to the global optimum?

Suppose two overlapping intervals exist.

    A = [2,5]
    B = [2,3]

If we keep

    [2,5]

the next interval cannot start until

    5

If we keep

    [2,3]

the next interval may start at

    3

Keeping

    [2,3]

never removes opportunities.

In fact,

it creates **more** opportunities.

Therefore,

keeping the interval that finishes earliest is always safe.

This is the greedy-choice proof.

## Code
```java
public class Problem3 {


    static void main(String[] args) {
        int [][] interval ={{1,2},{2,3},{3,4},{1,3}};
        // edge case
        if(interval == null || interval.length==0){
            System.out.println("Nothing to remove "+0);
            return;
        }

        // Sorting in ascending order as per finish time
        for ( int i=0;i<interval.length-1;i++){
            for(int j=i+1;j<interval.length;j++){
                if(interval[i][1]>interval[j][1]){
                    int[] temp =interval[i];
                    interval[i]=interval[j];
                    interval[j]=temp;
                }
            }
        }

       //int [][] interval ={{1,2},{2,3},{1,3},{3,4}};
        int lastFinish = interval[0][1];
        int minimCount=0;
        for(int i=1;i<interval.length;i++){
            if(interval[i][0]<lastFinish){
                minimCount++;
                System.out.printf("Remove [%s,%s] \n", interval[i][0],interval[i][1]);

            }else{
                lastFinish=interval[i][1];
            }

        }

        System.out.println("minimum remove "+minimCount);

    }
}
```

Time Complexity O(n²) - using own sorting
Space complexity o(1) - using own sorting so no extra space created
