## Problem :Two Sum
--------
## Statement

Given:

nums = [2, 7, 11, 15]

target = 9

Return the indices of the two numbers whose sum equals the target.

Answer:

[0, 1]
--------
## Brute Force:

### Idea:  
Given an integer array and a target, return the indices of the two elements whose sum equals the target. 
Each input has exactly one solution, and the same element cannot be used twice.

For every element, compare it with all remaining elements to find a pair whose sum equals the target.

### Code:

```java
public class TwoSumBruteForce {

    static void main(String[] args) {
        int[] input ={2,7,11,15};
        int target =9;

        int[] output = twoSum(input,target);
        System.out.println(Arrays.toString(output));
    }

    public static int[] twoSum(int[] nums, int target) {
        if(nums == null || nums.length <2)
            return new int[]{};

        for(int i=0;i<nums.length;i++){
            for(int j=i+1;j<nums.length;j++){
                if(nums[j]== target - nums[i]){
                    return new int[]{i,j};
                }

            }
        }

        return new int[]{};
    }
}
```

Time complexity =O(n^2)
Space Complexity =O(1)

### why rejected:
Time Complexity = O(n²)

For large inputs, the number of comparisons grows rapidly,
making this approach inefficient.

## Optimal Solution

### Idea: 
While traversing the array once,
store each visited number and its index in a HashMap.

For every new number,
calculate the required complement
(target - currentElement).

If the complement already exists in the map,
return the stored index and current index.

### Dry Run

```
nums = [2,7,11,15]
target = 9

Iteration 1

Current = 2

Need = 7

Map = {}

Store

2 → 0

------------------

Iteration 2

Current = 7

Need = 2

Map

2 → 0

Found

Answer = [0,1]
```

### Data Structure used:
HashMap having key the input value and value should be index.

### why this data structure used:

HashMap stores

Value → Index

allowing both constant-time lookup and retrieval O(1) of the required index.

### Code:

```java
public class TwoSumOptimizedUsingHashMap {

    static void main(String[] args) {
        int[] input={2,7,11,15};
        int target =9;

        int[] output = twoSum(input,target);
        System.out.println(Arrays.toString(output));
    }

    static int[] twoSum(int[] nums, int target) {

        if(nums == null || nums.length <2)
            return new int[]{};

        Map<Integer,Integer> map = new HashMap<>();
        for(int i=0;i<nums.length;i++){
            if(map.containsKey(target-nums[i]))
                return new int[] {map.get(target-nums[i]),i};
            map.put(nums[i],i);
        }
        return new int[]{};
    }
}

```

Time Complexity O(n)
Space Complexity O(n)

### Interview questions:
1. Why do we check containsKey() before put()?

If we inserted first and checked later, we'd overwrite the first index and could produce incorrect behavior.

Example : nums = [3, 3]   target = 6

2. Why do we use a HashMap instead of a HashSet?

We use a HashMap because the problem requires returning the indices of the elements, not just checking whether a value exists. A HashSet can only tell us if a value is present, whereas a HashMap stores the value along with its index, allowing us to return the correct answer.

### Common Mistakes:
1. Use Interfaces

Instead of HashMap<Integer,Integer> map = new HashMap<>();

Map<Integer,Integer> map = new HashMap<>();
2. Edge condition length <2 since two sum cannt be calculated if input array have length 1
3. Don't insert into the HashMap before checking containsKey().
4. Don't use nested loops after choosing HashMap, otherwise the complexity becomes O(n²).

## Pattern Recognition

### Pattern

HashMap Lookup

### Recognition

Whenever the problem asks

* Find a pair
* Find a complement
* Sum equals target
* Difference equals target
* Lookup previous elements

Think:

HashMap

instead of nested loops.
