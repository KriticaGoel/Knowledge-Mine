## Problem : Remove Duplicates from Sorted Array.
---
## Statement:

Given a sorted integer array,

nums = [1,1,2]

Modify the array in-place so that each unique element appears only once.

Return the number of unique elements.

Example:
```
Input

[1,1,2]

Output

2

Array becomes

[1,2,_]
```

### Constraints:
* Modify the input array in-place.
* Do not allocate another array.
* Return the count of unique elements.
---
## Brute Force:

### Idea:
Scan the array until the first duplicate is found.
Use its position as the write index and overwrite subsequent duplicates with the next unique elements.

### Code:

```java
public class removeDuplicatesBruteForce {

    static void main(String[] args) {
        int[] input ={1,2,2,3,3,4,4};

        System.out.println("unique Elements are "+removeDuplicate(input));
    }

    static int removeDuplicate(int[] nums) {

        if(nums==null || nums.length==0){
            return 0;
        }
        int write=-1;
        int unique=1;
        int j=0;
        for(int i=0;i<nums.length-1;i++){
            j=i+1;
            if(nums[i]==nums[j] && write==-1){
                write=j;
            }
            if(nums[i]!=nums[j]){
                if(write!=-1) {
                    nums[write] = nums[j];
                    write++;
                }
                unique++;
            }
        }
        System.out.println(Arrays.toString(nums));
        return unique;

    }
}
```


Time complexity: O(n)

Space Complexity :O(1)

### why rejected:
Although the algorithm runs in O(n),
it introduces unnecessary state management.

* write has multiple meanings
* requires special handling for the first duplicate
* more conditional logic
* harder to reason about and maintain

The optimal solution maintains one simple invariant:
write always points to the last unique element.
---

## Optimal Solution

### Idea: 
Maintain two pointers:

- read scans every element.
- write always points to the last unique element.

Whenever read finds a new unique value,
advance write and copy the value.

After completion,
the first (write + 1) elements contain all unique values.

### Data Structure used: 
Array

### why this data structure used:
The problem explicitly requires modifying the given array in-place,
so no additional data structure is used.

### Algorithm:
```
Initialize

write = 0

Loop

read = 1 → n-1

↓

If nums[read] != nums[write]

↓

write++

↓

nums[write] = nums[read]

↓

Return write + 1
```

### Code:
```
public class RemoveDuplicatesOptimal {

    static void main(String[] args) {
        int[] input ={1,2,2,3,3,4,4};

        System.out.println("unique Elements are "+removeDuplicate(input));
    }

    static int removeDuplicate(int[] nums) {
        if(nums==null || nums.length<2)
            return nums==null ? 0 : nums.length;
        int write =0;
        for (int read=1;read<nums.length;read++){
            if(nums[read]!=nums[write]){
                write++;
                nums[write]=nums[read];
            }
        }
        System.out.println(Arrays.toString(nums));
        return write+1;
    }
}
```

Time complexity  O(n)

Space Complexity  O(1)

### Interview questions:


### 1.
Why does this algorithm work only because the array is sorted?

Answer:
Duplicates are adjacent in a sorted array.
Therefore, comparing the current element with the last unique element is sufficient.

---

### 2.
Why are two pointers needed?

Answer:
One pointer scans every element,
while the other maintains the position where the next unique element should be placed.

---

### 3.
Why don't we clean the remaining array?

Answer:
The problem only requires the first k elements to contain the unique values.
The remaining elements are ignored.

---

### 4.
What happens if the array is not sorted?

Answer:
This algorithm fails because duplicates may appear anywhere.
A HashSet or sorting would be required.

## Common Mistakes

1.
Initializing write as -1.

2.
Forgetting to return write + 1.

3.
Comparing nums[read] with nums[read-1]
instead of nums[write].

4.
Trying to delete elements
instead of overwriting them.

5.
Using an extra array,
violating the in-place requirement.

---
## Pattern Recognition

### Pattern

Two Pointer

### Recognition

Think of this pattern when:

- The array is sorted.
- The problem asks to modify the array in-place.
- You need to remove duplicates.
- You need to compact valid elements.
- You need O(1) extra space.
