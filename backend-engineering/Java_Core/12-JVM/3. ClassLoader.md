1. **Loading** - Class loader load the class Hello.class by searching the classpath and load the bytes code into memory using JVM
   and generate class Object.

Class object contain metadata of the class like class name(Hello), names of super class and its also include method name.

This class object created only once when first time class loaded and later calls class loader fetch the data from memory and use it.

2. **Verification** - Check whether input class is well-formed or not using bytecode verifier? If not the reject the class

3. **Preparation** - If class pass the check, JVM allocate space for a static variables
4. **Resolution** - Loaded class (Hello.class) might be referencing other class.it could be accessing variable or method of another classes or
   creating instances of another classes. so JVM **load all the referenced classes**
5. **Initialization** - At point 3 we allocate the space of statice variables with default value. now JVM initialize the variables with
   actual user define values; they can be part of either the declaration or they could be part of static initializer block
   or instance initializer blocks


1. Super class will be loaded and initialized before the current class
2. In case of class its loaded and initialized.
3. In case of interface its loaded only not initialized.
4. Interface initialized
    1. Only if one of the static method is accessed
    2. Field initialized via method

Class loader locate Hello.class and load to memory.

There are three types of class loader
1. Bootstrap class loader - load core java api class also called as primordial class loader. it is implemented as part of JVM itself
2. Extension class Loader - load classes from the standard java extension api
3. Application/System class loader - load application classes from the classpath

Bytecode Verifier checks the bytecode for security and correctness


### First time loading of class

*  new instance is created
*  Invoking static method
*  Accessing static field
   *  Exception - Compile-time constants
* Subclass is loaded
* Run from command line
* Reflection

### First time loading of interface
*  Invoking static method
*  Accessing static field
   *  Exception - Compile-time constants
* Subclass or sub interface is loaded
* Run from command line
* Reflection

### Class Objects

All the objects that we create from a  particular class is always done using the class object of the class.

Class objects created for Class, Interface, Primitives, Void, Arrays

Class objects contains meta-information of the class.
1. String getName();
2. Class getSuperClass();
3. boolean isInterface();
4. Class[] getInterface();
5. ClassLoader getClassLoader();
