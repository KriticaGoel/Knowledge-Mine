# Interview Questions

---

# Beginner

<details>
<summary>What is Singleton Pattern?</summary>

A creational design pattern ensuring only one object exists.

</details>

---

# Intermediate

<details>
<summary>Why constructor is private?</summary>

To prevent outside object creation.

</details>

<details>
<summary>Why object is static?</summary>

Because singleton object belongs to class level.

</details>

<details>
<summary>Why getter method is static?</summary>

To access singleton without object creation.

</details>

---

# Advanced

<details>
<summary>How to make Singleton thread safe?</summary>

- synchronized
- double-check locking
- enum singleton

</details>

<details>
<summary>Why Enum Singleton is best?</summary>

- reflection safe
- serialization safe
- thread safe

</details>

<details>
<summary>What are disadvantages of Singleton?</summary>

- global state
- difficult testing
- tight coupling

</details>
